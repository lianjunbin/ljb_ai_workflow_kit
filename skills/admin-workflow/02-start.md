# /admin:start - 需求准备

**引导式信息收集**：通过交互式问答，逐步收集 Jira 需求、API 文档、业务上下文，最终生成符合 OpenSpec 规范的提案文件。

---

## 阶段进度

```
📍 当前进度
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ✅ INIT → 🔵 START → ⚪ DESIGN → ⚪ AUDIT → ⚪ APPLY → ⚪ REVIEW → ⚪ ARCHIVE
              ↑
           你在这里
```

## 输出文件结构（OpenSpec 规范）

```
openspec/changes/<change-id>/
├── proposal.md      # 提案文档（整合所有收集的信息）
│   ├── 需求概述（来自 Jira）
│   ├── API 接口（来自 Apifox）
│   ├── 业务分析（AI 分析 + 用户补充）
│   └── 复杂度评估
├── design.md        # 设计方案（DESIGN 阶段生成）
└── tasks.md         # 任务列表（DESIGN 阶段生成）
```

### 1.3 获取需求

使用 curl 直接调用 Jira REST API：

```bash
curl -s --user "<EMAIL>:<API_TOKEN>" \
  "https://<SITE>.atlassian.net/rest/api/3/issue/<ISSUE_KEY>" \
  | jq '{
    key: .key,
    summary: .fields.summary,
    description: .fields.description,
    status: .fields.status.name,
    priority: .fields.priority.name,
    assignee: .fields.assignee.displayName,
    labels: .fields.labels,
    acceptanceCriteria: .fields.customfield_xxxxx
  }'
```

   3. 或切换到「手动描述」方式

A: 可以选择「手动描述」方式，或者跳过 API 收集步骤
