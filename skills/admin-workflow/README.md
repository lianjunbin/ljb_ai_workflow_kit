# Admin Workflow Skills

管理后台 AI 开发工作流技能集，按阶段顺序组织。

## 目录结构

```
skills/admin-workflow/
├── 00-admin.md       # 工作流入口 - 展示全貌
├── 01-init.md        # 初始化项目配置
├── 02-start.md       # 需求准备（四步流程 · 多轮迭代）
│                     #   ① Jira获取(curl) ② API收集(Playwright/MD)
│                     #   ③ 业务分析 ④ 复杂度评估
├── 03-design.md      # 智能评估 + 提案设计 ← 含权限配置
├── 04-audit.md       # 提案审阅
├── 05-apply.md       # 应用代码
├── 06-review.md      # 代码走读
├── 07-archive.md     # 归档提案
└── other/            # 辅助文件
    ├── coding-standards.md  # 编码规范
    └── style-guide.md       # 样式指南
```

## 工作流阶段（7 阶段精简版）

```
┌─────────────────────────────────────────────────────────────────────┐
│       分析设计阶段（只读）          │      执行阶段（写入）         │
├─────────────────────────────────────────────────────────────────────┤
│ 01-INIT → 02-START → 03-DESIGN     │  05-APPLY → 06-REVIEW         │
│           → 04-AUDIT               │            → 07-ARCHIVE       │
├─────────────────────────────────────────────────────────────────────┤
│ ✅ 读取代码    ❌ 禁止写入          │  ✅ 写入代码                   │
│ ✅ 分析结构    ❌ 禁止修改          │  ✅ 创建/修改文件              │
│ ✅ 生成文档                         │  ✅ 执行命令                   │
└─────────────────────────────────────────────────────────────────────┘
```

## 快速开始

```bash
# 1. 初始化项目（首次使用）
/admin:init

# 2. 需求准备（一站式：Jira + API + 业务梳理）
/admin:start JIRA-123

# 3. 设计方案（含权限配置）
/admin:design

# 4. 审阅提案
/admin:audit

# 5. 应用代码
/admin:apply

# 6. 代码走读
/admin:review

# 7. 归档
/admin:archive
```

## v1.4 更新

- **阶段精简 9 → 7**：合并 START + API + BEFORE 为一个 START 阶段
- **一站式需求准备**：`/admin:start` 智能整合需求获取、API 收集、业务梳理
- **灵活执行深度**：根据需求复杂度自动调整，支持 `--quick` 和 `--full` 模式

## 设计原则

> **核心原则：APPLY 之前，只做分析设计，不动一行代码**
