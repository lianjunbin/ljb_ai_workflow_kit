
**状态图标：**
- ✅ 已完成
- 🔵 进行中
- ⚪ 未开始
- ⏸️ 已暂停
- ❌ 失败

## 3. 阶段卡片

**使用场景：** 每个阶段开始时展示说明卡片

### 模板

```
┌─────────────────────────────────────────────────────────────────┐
│  {emoji} {阶段名} · {阶段简述}                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  📝 这一步做什么？                                               │
│     {白话文描述，让用户秒懂}                                      │
│                                                                 │
│  ⏱️ 预计耗时：{时间}                                             │
│                                                                 │
│  💡 小贴士：                                                     │
│     • {提示1}                                                   │
│     • {提示2}                                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 各阶段 Emoji

| 阶段 | Emoji | 含义 |
|------|-------|------|
| INIT | 🎬 | 开始、初始化 |
| START | 📋 | 需求、文档 |
| API | 📡 | 接口、文档 |
| BEFORE | 🗺️ | 业务、梳理 |
| DESIGN | 🎨 | 设计、创作 |
| AUDIT | 📑 | 审阅、预览 |
| APPLY | ⚡ | 执行、应用 |
| REVIEW | 🔍 | 走读、优化 |
| ARCHIVE | 📦 | 归档、收纳 |

## 10. 语气规范

### 应该使用

- ✅ 白话文，通俗易懂
- ✅ 第二人称"你"
- ✅ 积极正面的表达
- ✅ 具体的操作指引

### 避免使用

- ❌ 过于技术化的术语
- ❌ 冷冰冰的系统消息
- ❌ 模糊的描述
- ❌ 负面消极的表达

### 示例对比

| ❌ 不好的 | ✅ 好的 |
|----------|---------|
| Error: PRD not found | 找不到需求文档，试试先运行 /admin:start |
| Executing task... | 正在帮你写代码，稍等一下... |
| Process completed | 搞定了！来看看改了什么 |
| Invalid parameter | 这个参数不太对，正确的用法是... |

| 信息 | 蓝色 | `\033[0;34m` |
