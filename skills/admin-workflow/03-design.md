```

     • 筛选条件变化后是否重置分页？

  ┌──────────────────────────────────────────────────────────────┐
```

**权限检测规则：**

```
自动检测触发条件:
  ✅ 新增页面/路由 → 生成路由权限 + 菜单配置
  ✅ 新增接口调用 → 生成接口权限
  ✅ 新增操作按钮（增删改导出等）→ 生成按钮权限
  ✅ 新增 Tab 页面 → 检查是否需要独立权限

跳过条件:
  ⏭️ 纯 UI 调整（样式、布局）
  ⏭️ 代码重构（不改变功能）
  ⏭️ Bug 修复（不新增功能）
```

## 快捷参数

**如果已经知道要什么模式，可以直接带参数跳过选择：**

| 参数 | 模式 | 说明 |
|------|------|------|
| `/admin:design` | 交互选择 | 显示上方菜单 |
| `/admin:design quick` 或 `-q` | 🚀 快速 | 1 个方案 |
| `/admin:design standard` 或 `-s` | ⭐ 标准 | 2 个方案（推荐） |
| `/admin:design deep` 或 `-d` | 💎 深度 | 3 个方案 |

**附加选项：**

| 参数 | 说明 |
|------|------|
| `-a` 或 `--auto` | 自动选择推荐方案 |
| `-r` 或 `--revise` | 修改现有提案 |

**组合示例：**

```bash
/admin:design               # 交互选择
/admin:design quick         # 快速（1 个方案）
/admin:design -q            # 快速（简写）
/admin:design standard      # 标准（2 个方案）
/admin:design -s            # 标准（简写）
/admin:design deep          # 深度（3 个方案）
/admin:design -d            # 深度（简写）
/admin:design -q -a         # 快速 + 自动选择（最快）
/admin:design -r            # 修改现有提案
```
